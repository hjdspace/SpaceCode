# 使用原则（5 条）

1. **先判断页面家族，再决定是否直接转换。** 这不是任意 HTML 通吃型转换器；先看它是不是已支持的页面家族（当前主要是 `v9-architecture` 和 `ai-runtime-page`）。

2. **没有 preset 的新结构，先补 preset，再跑生成。** 如果 HTML 的信息架构、模块分布、卡片/芯片/判断链等结构与现有 preset 不同，不要硬转；先补 DOM 提取规则、布局映射规则和 QA 规则。

3. **优先追求“可编辑 + 稳定”，不要追求任意网页像截图一样 100% 还原。** 这个 skill 的目标是把演示稿型 HTML 变成可编辑 PPT，而不是做浏览器级渲染复刻。

4. **生成不是结束，必须跑 QA。** 任何产物都应至少过一轮预审，重点检查文本溢出、标题压副标题、chip/卡片裁切、堆叠过满、间距不足等常见问题。

5. **能复用就复用，不能复用再扩展。** 遇到新页面时，先按 `references/preset-decision-rules.md` 判断能否作为现有 preset 的变体处理；只有当页面家族真的不同，才新增 preset，避免把 skill 做成大量一次性特例。
