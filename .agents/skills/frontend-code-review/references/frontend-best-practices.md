# 前端开发最佳实践手册

## 代码规范
1. 统一使用ESLint + Prettier + Stylelint做代码格式化和规范检查
2. 变量/函数名使用小驼峰，组件名使用大驼峰，常量使用全大写下划线分隔
3. 避免使用魔法数字，统一抽离为常量
4. 函数参数不超过3个，超过时使用对象传递
5. 避免嵌套超过3层，提前return减少嵌套

## React/Vue通用规范
1. 组件遵循单一职责原则，每个组件只做一件事
2. 避免编写超大组件，超过200行考虑拆分
3. 避免Props Drilling，使用状态管理或Context解决
4. 不要直接修改Props，状态提升到父组件管理
5. 列表渲染必须绑定稳定的key，禁止使用index作为key

## TypeScript规范
1. 禁止滥用any类型，尽可能定义具体类型
2. 类型定义抽离到单独的types目录或文件
3. 使用interface定义对象类型，type定义联合类型/交叉类型
4. 避免类型断言，除非你确定类型比TS推断更准确
5. 公共组件必须定义完整的Props类型

## 安全规范
1. 禁止直接使用innerHTML/eval，防止XSS攻击
2. 用户输入内容必须做转义处理
3. 敏感信息禁止硬编码在前端代码中
4. 第三方依赖定期升级，修复已知安全漏洞
5. 接口请求添加CSRF token校验

## 性能规范
1. 图片使用webp/avif格式，开启懒加载
2. 大列表使用虚拟滚动
3. 防抖节流处理高频事件
4. 路由和非首屏组件懒加载
5. 避免不必要的重渲染
