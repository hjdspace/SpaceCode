优化前 vs 优化后

核心思路：卡片聚焦"能做什么"，技术细节按需披露。

优化前的问题

  • 直接显示 officecli-pptx、html-slide-to-pptx 等内部技能 ID，用户看不懂
  • 技能数量 0–4 不等，卡片高度参差不齐，网格节奏破碎
  • 名称、描述、格式标签、技能徽章四类信息平铺，无主次
  • 后端已解析 skillDescriptions 人可读描述，但画廊从未使用
  • cowork（4 徽章）和 ppt-office-assistant（3 徽章）明显比其他卡片臃肿

优化后的改进

  • 移除技能 ID 徽章，卡片只留头像、名称、描述、格式标签——信息即决策
  • 所有卡片等高（min-height 132px），网格视觉节奏统一
  • 顶部 3px 类别色条替代文字类别标签，色彩暗示而非文字堆砌
  • 悬停展开浮层：人可读的能力描述 + 推荐提示词，渐进式披露
  • 底部能力指示点：极简暗示"有 N 项能力"，不暴露技术实现

减法优先

用户选助手时关心的是"它能帮我做什么"，而非"它调用了哪个 skill"。移除技术标识符，降低认知负荷。

渐进式披露

概览层只展示名称+描述+格式；悬停时浮层展开能力详情与推荐用法。信息按需呈现，不打扰浏览。

等高网格

统一 min-height + flex 布局，无论 0 个还是 4 个技能，卡片视觉一致。底部能力点用对齐留白填充。
