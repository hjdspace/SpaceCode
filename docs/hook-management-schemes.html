基础

Claude Code Hook 机制解析

Hook 是在 Claude Code 会话生命周期中特定事件触发时自动执行的自定义 Shell 命令。它们通过 settings.json 配置，支持 matcher 过滤和多种输出控制。

⚙ 配置结构

{ "hooks": { "PreToolUse": [ { "matcher": "Write|Edit", "hooks": [ { "type": "command", "command": "prettier --write $FILE_PATH", "timeout": 30 } ] } ], "Stop": [ { "hooks": [ { "type": "prompt", "prompt": "Check if all tasks complete: $ARGUMENTS" } ] } ] } }

⚡ 10 种 Hook 事件

事件触发时机支持 Matcher典型用途
PreToolUse工具执行前是拦截危险命令、验证路径
PostToolUse工具执行后是自动格式化、日志记录

对比

方案对比与推荐

四种方案各有侧重，根据用户群体和技术复杂度选择。

推荐
A

卡片列表

与 MCP 设置风格一致
  • 与现有 UI 风格完全统一
  • 开发成本最低，复用 MCP 组件
  • 按事件分组，逻辑清晰
  • 支持开关、展开详情
  • 适合大多数用户
B

事件时间线

生命周期可视化
  • 直观展示执行顺序
  • 帮助理解 Hook 触发流程
  • 适合学习和调试场景
  • 新增 Hook 时事件定位明确
  • 纵向空间占用较多
C

可视化构建器

拖拽式零代码配置
  • 新手友好，无需懂 JSON
  • 模板快速创建
  • 开发成本最高
  • 需要拖拽库支持
  • 高级用户可能觉得效率低
D

紧凑表格

高密度信息管理
  • 信息密度最高
  • 支持搜索和多维筛选
  • 适合 Hook 数量多的场景
  • 批量操作效率高
  • 新手可能感到信息过载

💡 推荐策略：A + D 混合模式

默认使用方案 A（卡片列表）作为主界面，与现有 MCP 设置风格一致，降低学习成本。 在卡片列表顶部提供「表格视图」切换按钮，高级用户可一键切换到方案 D 的紧凑表格模式。 方案 B 的时间线视图可作为调试模式的辅助展示。方案 C 的可视化构建器可作为未来增强， 在 V2 版本中引入。

创建 Hook 模态框设计

统一的创建/编辑 Hook 模态框，支持 command 和 prompt 两种类型，包含事件选择、matcher 配置、作用域选择等完整字段。

创建 Hook
选择 Hook 触发的生命周期事件
仅 PreToolUse / PostToolUse / PostCustomToolCall 支持 matcher。支持正则：Edit|Write 或 Notebook.*
可用环境变量：$CLAUDE_PROJECT_DIR, $FILE_PATH, $TOOL_NAME 等
使用 $ARGUMENTS 作为 Hook 输入 JSON 的占位符。LLM 将返回 approve/block 决策。

📝 删除 Hook 的交互设计

步骤交互说明
1. 触发点击删除按钮卡片/表格行上的删除图标
2. 确认弹出确认对话框显示 Hook 名称和命令，防止误删
3. 执行调用 IPC 删除从对应 settings.json 中移除条目
4. 反馈Toast 通知「Hook 已删除」+ 撤销按钮（5秒内）
5. 撤销点击撤销恢复刚删除的 Hook 条目
