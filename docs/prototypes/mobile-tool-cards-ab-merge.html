Mobile App 工具卡片 · A+B 融合方案

融合方案：A 的 Anthropic 暖色气质（卡片圆角、配色跟随主题）+ B 的开发者技术感（macOS 红黄绿窗口控件作为 run_command 标识、GitHub 风格 diff、JetBrains Mono 全等宽）。所有颜色从现有 SpaceCodeTheme._ColorSet 取，代码高亮色从 CodeTheme 取，4 套主题下完美适配。

设计要点（Flutter 实现指引）：
· 主题色源：卡片背景、边框、文字色全部从 Theme.of(context).colorScheme + SpaceCodeTheme._ColorSet 取，不写死 hex。代码高亮色从 CodeTheme.light/dark/anthropic/anthropicDark 匹配当前主题取。
· 终端永远深色：run_command 卡片的终端区域无论浅色/深色主题都用深色底（终端模拟器惯例），但 prompt 色跟随 accentPrimary（light=青绿、dark=蓝、anthropic=橙、anthropicDark=橙）。
· macOS 窗口控件：所有 run_command 卡片头部固定红黄绿三色点（#ff5f56 / #ffbd2e / #27c93f），不随主题变——这是开发者识别符号，类似 VSCode 终端。
· diff 红绿背景：删除行 error.withOpacity(0.12)，新增行 success.withOpacity(0.15)；浅色主题用深红/深绿字，深色主题用浅红/浅绿字（自动跟随 textPrimary 亮度）。
· WRITE 左色条：卡片左侧 3px 绿色边 (success) + 头部 + ADDED 标签 + 代码区淡绿底 success.withOpacity(0.04)。
· READ 代码块：浅色主题用浅色代码底（如 GitHub Light），深色主题用深色代码底；行号色 textMuted.withOpacity(0.5)。
· ToolCall model 扩展：需新增结构化字段 toolKind（run_command/read/write/edit/search/other）、filePath、language、diffHunks（List<DiffLine>）、terminalOutput。头部渲染根据 toolKind 分发到对应子 widget。
