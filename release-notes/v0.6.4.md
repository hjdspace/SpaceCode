## SpaceCode v0.6.4

本版本重点完成 Turn store 深化重构，将回合生命周期、权限裁决、事件订阅从 chat.ts Proxy 迁移至独立的 Turn 模块；同时整合了设计模式预览功能与 H5 引擎事件路由。

## Highlights

- **Turn store 深化**：新增 Turn store 骨架与 SessionSink 写回接口，迁移 TurnState 状态机（beginTurn/ensureTurn/endTurn/resetTimeout）、Engine 事件订阅与 handler、消息收发（send/abort/retry/submitToolAnswer/skipToolAnswer）、权限裁决（pendingPermissions/allow/deny + 订阅）以及 pending-messages 与 auto-retry 状态机，并显式 bootstrap 初始化顺序（ADR-0003）。
- **权限模式策略 store**：抽出独立的 permissionPolicy store，分离权限模式管理与 Turn 生命周期。
- **AskUserQuestion 摘要视图**：为 AskUserQuestion 工具新增问题摘要展示视图。
- **H5 模式适配**：新增 H5 模式适配与引擎事件路由功能。
- **设计预览工作区**：新增设计预览功能与 DesignFileWorkspace 组件，集成至 InfoPanel，简化 DesignPage 布局。

## Fixes

- 修复会话加载状态异常和进程退出逻辑问题。
- 删除 chat 残留 Proxy，改为纯 re-export。
- 修复 turn 测试 fake api 返回值并补回不变量注释。
- 避免重复渲染权限请求卡片并修复参数泄露问题。
- 修复 H5 端会话消息被覆盖的问题。
- 修复生产模式下 designResourcesPath 路径问题。

## Refactor

- 删除已废弃的 chatStream.ts 与 chatControl.ts，移除 chat.ts Proxy，消费方按需 import turn/session/policy store。
- 用 createSettledTurn() 替换不安全的 as TurnState 断言。
- 重构 design-panel 文件监听逻辑，优化组件职责划分。
- 删除 DesignSplitView，简化 DesignPage 布局。
- 优化 TemplatePicker、DesignSystemPicker、PermissionModeSelector 样式。
- ChatInput 响应式设计优化。
