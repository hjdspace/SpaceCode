## SpaceCode v0.6.8

移动端 AI Agent 全功能上线：本地 Agent 支持 SSE 流式输出和工具调用，GitHub 集成实现 Device Flow 认证、自动 clone 仓库与 PR 创建。

## Highlights

- **本地 Agent 流式输出**：LocalAgentService 重写为 SSE 流式接口，LLM 回复实时逐 token 显示；支持 list_files / read_file / write_file 工具调用循环（最多 8 轮），自动完成代码读写
- **GitHub Device Flow 认证**：手机端通过浏览器完成 GitHub 授权，弹窗 polling 自动关闭，认证 token 持久化到本地
- **GitHub 仓库自动 clone 与 PR**：选择 Github 仓库后自动 clone 到手机工作目录；Agent 修改文件后自动 commit 并创建 Pull Request
- **工作栏 UI 优化**：本地目录与 Github 仓库入口分离，会话级工作目录切换
- **模型列表 API 拉取**：设置页支持从 OpenAI 兼容 `/models` 接口拉取模型列表，下拉选择或手动输入
- **ToolCall 状态实时反馈**：Github clone 和 PR 创建以虚拟 ToolCallCard 形式显示运行中/已完成状态
- **配置持久化**：应用启动时预加载 MobileConfig，修复重启后 API Key 丢失需重新点保存的问题
- **多会话并行管理**：新增权限模式设置，支持多会话并行工作

## Fixes

- 修复 Device Flow 认证完成后弹窗不自动关闭的问题
- 修复聊天工具栏第二个 chip 与本地模式重复，无法选择 Github 仓库
- 移除 system prompt 中 "受 Pi 启发" 等无关品牌内容
- 修复流式聊天消息渲染时序问题

## Build

- CI 新增 `build-android` job，自动构建 release APK 并上传到 GitHub Release（产物名 `spacecode-mobile-v0.2.0.apk`）
- 修复 file_picker 8.3.7 与 AGP 9 的 compileSdk 兼容性问题（CI 中自动 patch Pub Cache）
